分析这段 Python 代码是否存在SQL注入风险：{' '}
def get_user_data(user_id):

query = f{'"'}SELECT * FROM users WHERE id = {'{'}user_id{'}'}{'"'}
result = db.execute(query)
return result
在您的代码中发现 2 处潜在 SQL 注入漏洞：

1. 'query' 变量直接拼接了未经验证的 user_id 参数
2. 使用字符串格式化构建SQL查询非常危险

建议修复方案：
def get_user_data(user_id):

query = "SELECT * FROM users WHERE id = ?"
result = db.execute(query, (user_id,))
return result
这样使用参数化查询可以有效防止SQL注入攻击。
